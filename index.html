香港謎路 · 回憶機解謎 Web App

完整的 Development Brief 與功能設計規格 (MVP 版本)

1. 整體目標 (Overall Objective)

建立一個 **Mobile-first Web App**，玩家以「回憶機解碼師」身份，在真實街區完成 Geo-location 任務和解謎，解鎖劇情與回憶。MVP 階段聚焦於**單一路線 (大角咀 · 爺爺的回憶)**，重點在於建立穩定的遊戲流程與回憶歸檔系統。

2. 故事背景 (Narrative Premise)

「某位客人把一部裝載家族記憶的『回憶機』交給你，這台機器已上鎖。你需要走入大角咀舊街區，透過環境線索收集回憶鎖匙，拼接出爺爺隱藏一生的願望。」
  • 任務一： 到訪回憶地點，通過現場觀察完成解謎，收集「回憶鎖匙」。
  • 任務二： 在回憶機整合碎片，推論出「爺爺的遺願」，完成 Final Mission。

3. 遊戲流程 (Detailed Flow)

I. Onboarding 進場

Landing Page (輸入電話) → Consent Page (同意定位與私隱條款) → User Guide (玩法與安全提示) → Map Select (選擇大角咀路線)。

II. Exploration 探索

World Map (顯示玩家與謎題位置) → Geofencing (進入 30m 半徑觸發) → Puzzle Screen (現場解謎)。

III. Interaction 互動

導入 **Drag & Drop (拖放)** 功能，模擬拼湊相片、修復物件，增加實體操作感而非純文字選擇。

IV. Memory Archiving 歸檔

回憶機 Subpage (獨立子頁) 展示九宮格存檔。玩家隨時隨地可閱讀已解鎖的長篇劇情。

4. 技術細節 (Tech Specs)

Framework Next.js 14 (App Router)
Styling Tailwind CSS
Interaction Framer Motion (Drag & Drop)
Storage Browser LocalStorage
Geolocation Native Browser API (watchPosition)
