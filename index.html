香港謎路 · 回憶機解謎
Development Brief (MVP)

1. 整體目標 (Overall Objective)

建立一個 Mobile-first Web App，玩家以「回憶機解碼師」身份，在真實街區完成 Geo-location 任務和解謎，解鎖劇情與回憶。MVP 階段聚焦於單一路線（例如：大角咀 · 爺爺的回憶），重點是清晰的遊戲流程、回憶機 Subpage 與資料結構，方便日後擴充。

2. 用戶角色與故事前提 (Narrative)

玩家身份：回憶機解碼師。

起始情境：某位客人把一部裝載家族記憶的「回憶機」交給玩家，請求協助解鎖。

任務與解鎖條件：

  • 任務一：到訪回憶中的地點，收集「回憶鎖匙」。取得足夠鎖匙後，開始解鎖回憶片段。
  • 任務二：整合所有已解鎖的回憶線索，找出「爺爺的遺願」，完成整體故事。

3. 登入／開局流程

LandingPage

核心 CTA：「輸入電話號碼開始」。表單只需要電話號碼（Text Input）。

ConsentPage (個人資料及定位同意)

需清楚說明收集資料類型（電話、進度、定位）及其用途（判斷抵達位置、人流統計）。提供必選的資料使用同意 Checkbox 及可選的推廣資訊 Checkbox。

UserGuide & MapSelect

顯示遊戲玩法概述與安全提示（注意交通、勿進私地）。MVP 僅顯示「大角咀 · 爺爺的回憶」路線。

4. 遊戲起始與主流程 (Game Flow)

WorldMap (地圖畫面)

顯示玩家位置與謎題標記（3–6 個）。使用 Geolocation API 判斷半徑（30–50 公尺），按鈕變為「開始解謎」。

PuzzleScreen (單一地點解謎)

包含地點背景、題目本體。MVP 支援：Multiple Choice (MCQ)、Short Text (填空) 及 Drag and Drop (拖放交互)。

5. 回憶機 (Memory Machine) Subpage

角色定位：獨立子頁，提供純閱讀模式。玩家在玩完後或在家中也可打開此頁面。顯示九宮格或卡片列表，點擊已解鎖格進入 Detail Page 閱讀長篇故事。

6. 最終任務 (Final Mission)

當收集滿 6 或 9 個鎖匙後，觸發 Final Story。玩家需根據回憶線索回答爺爺的遺願，決定 Ending 分支（好結局 / 遺憾結局）。

7. 技術要求 (Tech Stack)

  • Frontend: Next.js (App Router) + Tailwind CSS
  • Interaction: Framer Motion (Drag & Drop Support)
  • Storage: LocalStorage (Browser-based)
  • Map: Google Maps / Leaflet API
